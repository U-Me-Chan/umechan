# Umechan

Это проект, объединяющий в себе функционал анонимного текстового форума и интернет-радиостанции. Проект ориентирован на минималистичность и простоту развёртывания за счёт контейнеризации и скриптов автоматизации часто возникающих задач. По большей части ориентирован на частное использование людьми с некоторым техническим бекграундом и любовью к KISS-инструментам.

## Ключевые особенности
* Markdown-разметка (эталонная реализация веб-клиента к форуму использует слегка модифицируемый GFM)
* Вложения [BETA]: к посту можно крепить изображения или видео напрямую в форме создания поста или вручную через сниппет `[![](http://domain.tld/path/to/files/example.thumbnail.jpeg)](http://domain.tld/path/to/files/example.jpeg)`, на текущий момент поддерживается работа только с внутренним сервисом `chan-filestore`
* Темы раздела можно крепить к верху доски или блокировать к ответу
* Идентификация [BETA]: возможность резервировать имя автора поста и использовать его по ключу-паролю в поле "Автор"
* Эталонная реализация веб-клиента на Vue2 + Buefy доступна по адресу `http://domain.tld` после установки
* Конъюнкция досок на уровне API и в адресной строке эталонного клиента
* Возможность делать доски непубличными, исключая их из списка досок и ленты последних постов
* Управление и администрирование осуществляется с помощью `Makefile`-задач: бекапы, управление досками и темами, восстановление из бекапа
* Практически всё публичное REST API описано с помощью OpenAPI
* "Умная" система ротации очереди на радио: на каждый временной период существует схема, описывающая порядок и условия постановки композиций в очередь на основе их популярности и даты последнего воспроизведения
* Голосование за текущую композицию [BETA]: значение оценки учитывается при составлении очереди воспроизведения, композиции ниже порогового значения оценки в очередь не попадут
* Импорт композиций: отдельный сервис, добавляющий файлы из директории в БД радиостанции. Файлы валидируются по внутренним требованиям в виде пайплайна
* Доменные события в Kafka [ALPHA]: будущий механизм синхронизации данных между двумя нодами
* Бекапы постов и файлов с шифрованием симметричным ключом
* Кеширование ответов API форума с помощью Memcached [BETA]
* Сторонние клиенты к форуму: https://github.com/U-Me-Chan/pissychan-front и https://github.com/kugimiya/umechan-opti

## Установка (только форум)

0. Установите `docker`, `docker-compose`, `sqlite3`, `gpg`, `tar` и `make`.
1. Склонируйте репозиторий с проектом: `git clone https://github.com/U-Me-Chan/umechan.git`; перейдите туда: `cd umechan`
2. Скопируйте пример конфигурации проекта: `cp .env.dist .env`, отредактируйте следующие переменные окружения:
* `ADMNISTRATOR_KEY` - ключ для выполнения удалённых операций от лица владельца
* `DOMAIN` - публичный домен
* `DATABASE_BACKUP_DIR` - директория для зашифрованных ежедневных дампов БД
* `FILESTORE_SOURCE_DIR` - директория с загруженными файлами
* `FILESTORE_BACKUP_DIR` - директория для зашифрованных ежедневных архивов файлов
* `FILESTORE_BACKUP_INDEX_DB` - путь и имя файла индекса по бекапу файлов
* `BACKUP_PASSPHRASE` - парольная фраза для шифрования бекапов
3. Выполните команду `make`: она произведёт сборку необходимых образов, поднимет требуемые контейнеры и применит схемы миграций к БД
4. Создайте новую доску: `make chan-create-board tag=[foo] name=[Foo]`, замените текст с квадратными скобками по вкусу.
5. Откройте `http://domain.tld/board/foo` и сможете увидеть интерфейс просмотра доски.

## Команды управления и администрирования

* `make chan-set-[un]sticky-thread` - прикрепляет/открепляет тему от верха доски
* `make chan-set-[un]blocked-thread` - запрещает/разрешает ответы на тему
* `make chan-set-[un]public-board` - устанавливает/снимает флаг публичности доски
* `make chan-create-board` - создаёт новую доску
* `make chan-rebuild-thumbnails` - ребилдит *все* миниатюры загруженных файлов
* `make chan-backup-filestore` - выполняет бекап файлов и шифрует его симметричным ключом
* `make chan-backup-database` - выполняет бекап БД чана и радио, шифрует симметричным ключом
* `make radio-backup-music` - синхронизирует состояние текущей директории музыкальной библиотеки с директорией-бекапом

## Для разработчиков
Собираем бекенд:
```bash
make development
```

Запускаем дев-сервер клиентской части приложения:
```bash
cd frontend && npm i && npm run serve
```
